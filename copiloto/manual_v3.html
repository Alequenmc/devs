Sistema de Targeting

Cómo ejecutar comandos en múltiples grupos simultáneamente.

Local

Sin argumentos

Si no se usa --groups, el comando se ejecuta en el chat actual.

Específico
--groups=001,002

Ejecuta el comando solo en los grupos con ID corto 001 y 002.

Global
--groups=all

Ejecuta en todos los grupos Standard y Console. Excluye grupos 'sensitive'.

01 Registro de Grupos

local

!mod register <type>

Inicializa el bot en el grupo actual.

!mod register standard

!mod register console

!mod register sensitive

!mod unregister

Elimina el grupo actual de la base de datos. Detiene la moderación.

02 Consola: Gestión de Grupos

Comando Descripción Ejemplo
listgroups Muestra tabla de grupos registrados, sus IDs cortos y tipos. !mod listgroups
settype Cambia el tipo de grupo remotamente. !mod settype 002 console

03 Consola: Claves-Valores (!mod config)

Estas son las claves que puedes modificar con !mod config <clave> <valor>. (Ej: !mod config SPAM_STRIKE_LIMIT 5)

Clave (Key) Descripción Valor Típico
SPAM_STRIKE_LIMIT Nº de strikes (spam, flood) para ban automático. 3
SPAM_STRIKE_TIMEFRAME_MS Ventana de tiempo (en milisegundos) para el ban. 60000 (1 min)
USER_RATE_WINDOW_MS Ventana de tiempo para detectar flood. 12000 (12 seg)
USER_RATE_MAX_MSGS Nº de mensajes en esa ventana para ser flood. 6
MIN_MESSAGE_LENGTH_FOR_AI Longitud mínima para enviar a la IA (ahorra recursos). 14
USE_AI_FOR_MEDIA Analizar con IA mensajes que solo tienen media (foto/video) y texto corto. false
!mod config list

Usa este comando para ver la lista y sus valores actuales.

04 Consola: Base de Datos

Mantenimiento

  • !mod db status Muestra conteo de filas (strikes, grupos, mutes, etc).
  • !mod prune <type> <age> Borra datos antiguos.

Argumentos para PRUNE:

Tipos: strikes, offenses, rate_events
Edad: 30d, 24h, 60m (días, horas, minutos)
Ej: !mod prune strikes 30d

05 Consola: Whitelists & Blacklists

Dominios

!mod allow google.com
!mod deny sitio-spam.com

Frases (IA Bypass)

⚠️ REQUIERE COMILLAS DOBLES

!mod allowphrase "texto seguro"
!mod denyphrase "frase prohibida"

Consulta

!mod list <allowed|blocked|phrases>

06 Gestión Remota de Grupos

Comandos para administrar metadatos y permisos de grupos a distancia. Todos usan --groups.

Comando Soporta 'All' Uso / Argumentos
setsubject SÍ --groups=... "Nuevo Título"
Las comillas son recomendadas.
setdesc SÍ --groups=... "Nueva Descripción"
setsetting SÍ --groups=... <messages|edit> <all|admin>
Cierra chat (messages admin) o info (edit admin).
announce SÍ --groups=... <mensaje>
REQUIERE --groups obligatoriamente.
promote NO @user --groups=001,002
Convierte al usuario en Admin.
demote NO @user --groups=001,002
Quita permisos de Admin.

07 Acciones de Moderación

Acciones directas contra usuarios. Se pueden ejecutar localmente o con --groups.

!mod kick @user

Expulsa al usuario del grupo.

No soporta 'All'
Admite mención, ID (52..@c.us) o búsqueda.
!mod reset @user

Reinicia los strikes del usuario a 0.

No soporta 'All'
!mod mute @user <tiempo>

Silencia. Si habla, recibe strike FLOOD y se borra el mensaje.

!mod mute @Juan 30m --groups=all
Soporta 'All'
!mod unmute @user

Levanta el silencio manualmente antes de tiempo.

Soporta 'All'

08 Auditoría e Información

Comandos informativos. La mayoría solo devuelven datos de 1 grupo a la vez (no usar --groups=all).

!mod whois @user

Perfil completo: ID real, strikes, estado de mute y últimas 5 ofensas.

!mod status

Muestra límites de flood y strikes configurados en el grupo actual.

!mod mutes

Lista todos los usuarios actualmente silenciados en el grupo y tiempo restante.

!mod help

Envía una lista resumida de comandos al chat.

!mod strikes [@user]

Muestra strikes. Si no se menciona usuario, muestra el Top 20 del grupo.

Flags Avanzados:
  • --window=2h // Strikes en las últimas 2 horas
  • --all-time // Historial completo
